This repository contains the reproducibility materials and obtained results for the paper **"rudof-MCP: A Model Context Protocol Server for Semantic Web Operations"**, presented at **ESWC 2026** (Posters & Demos Track).

## 1. Overview

**rudof-MCP** is a [Model Context Protocol](https://modelcontextprotocol.io/) (MCP) server that exposes the capabilities of the [rudof](https://github.com/rudof-project/rudof) library (a Rust-based toolkit for Semantic Web operations) as tools that any MCP-compatible Large Language Model (LLM) client (e.g. Claude Desktop) can invoke directly.

The goal is to enable LLMs to delegate Semantic Web operations to a **deterministic, specification-compliant engine**, allowing them to reason over actual results rather than relying solely on pretrained knowledge.

## 2. Demonstration

## 3. Repository Content

### 3.1. Input materials (`/input`)

These files were used during the demonstration to showcase the server's capabilities:

| File | Purpose |
|---|---|
| `data.ttl` | Core RDF dataset in Turtle: it models a fictional organisation (`ex:Acme`) with employees and managers. |
| `query.sparql` | SPARQL `SELECT` query returning every `ex:Person` with their name and age, ordered by descending age. |
| `schema.shex` | ShEx schema for `Organization`, `Employee` and `Manager`. **Intentionally contains a negative cycle** (`ex:manages NOT @ex:EmployeeShape`), an invalid construct in ShEx. |
| `shapemap.sm` | Shape map pinning each instance (`Alice`, `Bob`, `Carol`, `Acme`) to the shape it should conform to. |
| `schema.ttl` | SHACL constraints requiring `ex:name` (string) and `ex:age` (integer) on every `ex:Person`, and `ex:name` on every `ex:Organization`. |

## 4. How to Reproduce

1. **Configure rudof-MCP in Claude Desktop.** For detailed step-by-step instructions, please refer to the [official documentation](https://rudof-project.github.io/rudof/cli_usage/mcp.html).
2. **Execution**: Use the files provided in the `/input` directory to prompt Claude.
